Boa — aqui vão opções práticas pra você testar CAPTCHAs no seu jogo, com um pequeno resumo do que cada uma serve e onde testar:

  1. Google reCAPTCHA (demo + chaves de teste) — tem uma página de demonstração onde você pode ver o widget em ação e a FAQ do Google fornece chaves de teste (que sempre passam e são próprias para ambientes de desenvolvimento). Útil se quiser testar o fluxo clássico “I'm not a robot” ou reCAPTCHA v2/v3. Google+1

  2. hCaptcha (demo & docs) — alternativa ao reCAPTCHA; oferece demo e documentação para gerar sitekeys/secret keys. Bom se quiser testar outra UX (checkbox, diferentes níveis de desafio). accounts.hcaptcha.com+1

  3. Cloudflare Turnstile (demo / keys de teste) — solução sem puzzles visuais (mais “transparente” para o usuário). Cloudflare fornece demos e orientações para teste (incluindo chaves dummy para desenvolvimento). Ótimo se você quer algo leve e com boa privacidade. Cloudflare+1

  4. Friendly Captcha (demo) — abordagem “privacy-first” e acessível; tem demo e exemplos (útil se quer evitar imagens/texto e priorizar acessibilidade). Friendly Captcha+1

Extras rápidos e dicas práticas

  • Para testes automatizados, use as chaves de teste oficiais quando existirem (por exemplo o reCAPTCHA v2 tem chaves de teste que fazem todas as verificações passarem). Não rode CAPTCHAs “reais” em ambientes de CI sem um modo de teste — vai travar seus testes. Google for Developers

  • Se você só precisa simular a presença de um CAPTCHA no jogo (sem validar tokens com servidores externos), considere usar um mock local do widget (um botão que retorna “ok”) durante desenvolvimento/testes automatizados — mais simples e sólido para QA.

  • Evite serviços que burlam/solvem CAPTCHAs automaticamente (p.ex. serviços de “captcha solving”) — além de potenciais problemas legais/termos de uso, não é necessário para testes internos e pode gerar mau funcionamento em produção. (Se precisar de usuários reais para testar, prefira testes manuais com colegas ou beta testers.)

Também explico como usar o arquivo localmente e ideias simples de uso na sua partida de RPG.

Como usar (resumo rápido)

  1. Copie todo o código abaixo e salve num arquivo chamado captcha-tester.html.

  2. Abra o arquivo no seu navegador (duplo-clique ou Ctrl+O e selecione o arquivo).

  3. Escolha o modo (Mock / reCAPTCHA / hCaptcha / Turnstile).

  4. Clique no botão para “Iniciar desafio” — o widget aparecerá (ou, no mock, simula um desafio).

  5. Para integrar no RPG: abra a página num navegador ou num dispositivo secundário. Quando o mestre usar a habilidade “desestabilizar IA”, ele precisa completar o CAPTCHA para evitar o efeito.

Observação: para hCaptcha e Turnstile em produção ou para testar chaves reais, você precisa criar sitekeys/secret keys nos dashboards respectivos. Para reCAPTCHA v2 existe uma chave de teste pública que sempre passa (útil pra desenvolvimento). Google for Developers+2docs.hcaptcha.com+2

<script>
  // reCAPTCHA v2 test keys (Google fornece chaves de teste que sempre passam para dev).
  // Fonte: documentação oficial reCAPTCHA (usar só para desenvolvimento). 
  const RECAPTCHA_TEST_SITEKEY = "6LeIxAcTAAAAAJcZVRqyHh71UMIEGNQ_MXjiZKhI";

  const modeEl = document.getElementById('mode');
  const widgetArea = document.getElementById('widgetArea');
  const statusDetail = document.getElementById('statusDetail');
  const hSiteKeyInput = document.getElementById('h_sitekey');
  const tSiteKeyInput = document.getElementById('t_sitekey');

  let currentWidget = null;

  function log(text){
    statusDetail.innerHTML = text;
  }

  document.getElementById('startBtn').addEventListener('click', () => {
    const mode = modeEl.value;
    widgetArea.innerHTML = '';
    log('Iniciando: ' + mode);
    if(mode === 'mock'){
      renderMock();
    } else if(mode === 'recaptcha'){
      renderRecaptcha();
    } else if(mode === 'hcaptcha'){
      renderHcaptcha();
    } else if(mode === 'turnstile'){
      renderTurnstile();
    }
  });

  document.getElementById('resetBtn').addEventListener('click', () => {
    widgetArea.innerHTML = '';
    log('Resetado.');
  });

  // 1) Mock: botão que simula um desafio (útil para mesa sem Internet)
  function renderMock(){
    const btn = document.createElement('button');
    btn.textContent = 'Resolver (Mock)';
    btn.onclick = () => {
      // simula 70% de chance de sucesso
      const success = Math.random() < 0.7;
      if(success){
        log('Mock: sucesso — IA estabilizada, sem dano.');
      } else {
        log('Mock: falha — IA aplicada debuff.');
      }
    };
    widgetArea.appendChild(btn);
  }

  // 2) reCAPTCHA v2 (usa a chave de teste do Google)
  function renderRecaptcha(){
    const wrapper = document.createElement('div');
    wrapper.innerHTML = `<div class="g-recaptcha" data-sitekey="${RECAPTCHA_TEST_SITEKEY}"></div>
                         <div style="margin-top:8px"><button id="verifyRec">Verificar token (simulado)</button></div>`;
    widgetArea.appendChild(wrapper);
    // O Google carrega automaticamente o widget pela tag script incluída no <head>.
    // A verificação de token normalmente é feita no servidor; aqui apenas simulamos.
    setTimeout(()=>{ log('reCAPTCHA renderizado (test key). Resolva o desafio e clique em "Verificar token".'); },300);
    wrapper.querySelector('#verifyRec').addEventListener('click', ()=>{
      // no ambiente de dev com a test key, as verificações costumam passar; aqui mostramos mensagem.
      log('Verificação (simulada): token aceito — resultado: sucesso.');
    });
  }

  // 3) hCaptcha (carrega via script; requer sitekey)
  function renderHcaptcha(){
    const sitekey = hSiteKeyInput.value.trim();
    if(!sitekey){
      widgetArea.innerHTML = '<div class="muted">Insira sua <strong>hCaptcha sitekey</strong> no campo acima antes de iniciar.</div>';
      return;

    const container = document.createElement('div');

    widgetArea.appendChild(container);
    setTimeout(()=>{ log('hCaptcha renderizado. Resolva o desafio e clique em "Verificar token".'); },300);

      log('Verificação (simulada): token recebido. Em ambiente real, envie token ao seu servidor para verificar com secret key.');
    });
  }

    widgetArea.appendChild(container);

      log('Verificação (simulada): token recebido. Em produção, seu servidor deve chamar o endpoint de verificação da Cloudflare.');

Notas importantes / links úteis

  • reCAPTCHA — o Google fornece chaves de teste para reCAPTCHA v2 que sempre passam (úteis para desenvolvimento). Não as use em produção; crie chaves no console do reCAPTCHA para seu domínio. Google for Developers+1

  • hCaptcha — crie um sitekey/secret no painel do hCaptcha (eles têm docs claros sobre como integrar o widget). docs.hcaptcha.com

  • Cloudflare Turnstile — possui documentação com testing keys e instruções de integração para desenvolvimento. Use as testing keys indicadas pela Cloudflare para testes locais. Cloudflare Docs+1

  • Friendly Captcha — alternativa focada em privacidade; também tem demo e API (caso queira testar outra abordagem). Friendly Captcha+1

Onde hospedar (opções simples)

  • Local: abra o arquivo direto no navegador (mais simples).

  • GitHub Pages: crie um repositório e faça upload do captcha-tester.html — fique disponível publicamente em <username>.github.io/<repo>/captcha-tester.html. (Se preferir, eu te dou os passos exatos para publicar.)

  • Netlify / Vercel: arrastar e soltar o arquivo no painel e já fica no ar — bom para testes rápidos.

Sugestões de uso para a sua cena de RPG (prático)

  • O mestre resolve o CAPTCHA no dispositivo dele quando o jogador ativa a habilidade; falha = recebe dano/debuff, sucesso = evita.

  • Alternativa: o mestre designado (ou um jogador adversário) resolve o CAPTCHA; isso dá um elemento divertido e tenso ao combate.

  • Use Mock para partidas offline — ele simula resultado rápido sem depender de internet.

🚀 Publicando no GitHub Pages

1. Criar conta no GitHub (se ainda não tiver)

  • Acesse https://github.com

  • Clique em Sign up e siga o cadastro.

2. Criar um novo repositório

  1. Depois de logado, clique no botão “+” no canto superior direito → New repository.

  2. Nome do repositório: por exemplo captcha-rpg.

  3. Marque como Public.

  4. Clique em Create repository.

3. Fazer upload do arquivo

  1. No repositório recém-criado, clique em “Add file” → “Upload files”.

  2. Arraste e solte o arquivo captcha-tester.html (aquele que você salvou com o código que te passei).

  3. Role até o fim e clique em Commit changes.

4. Ativar o GitHub Pages

  1. Vá até as Settings do repositório.

  2. No menu lateral, clique em Pages.

  3. Em “Source”, escolha a branch main e a pasta /(root).

  4. Clique em Save.

5. Acessar o site

  • Depois de alguns segundos, o GitHub vai gerar um link do tipo:

    arduino
    https://<seu-usuario>.github.io/captcha-rpg/captcha-tester.html

  • Esse link já é o site online.

  • Compartilhe com o grupo: cada um pode abrir no celular ou PC e usar na mesa.

🎲 Como usar no RPG

  • Quando a habilidade da IA for ativada, o mestre acessa a página.

  • Escolhe Mock (se quiser só sortear falha/sucesso) ou reCAPTCHA/hCaptcha/Turnstile se tiver internet.
